**System Task:** Update the "OnlyNotes" Next.js/Supabase project with three critical modules: Authentication, Administration, and Content Creation.

### 1. Module: Google Auth & User Management
**Stack:** Supabase Auth (Google Provider).
**Requirements:**
* **Login Screen:** Create a clean, modal-based login screen with a "Sign in with Google" button.
* **Database Sync (Crucial):**
    * Create a Supabase Database Trigger. When a user signs up via Auth, automatically insert a row into `public.profiles`.
    * Map `auth.users.email` and `auth.users.user_metadata.avatar_url` to the profile.
* **Profile Edit UI:** In the "Profile" tab (bottom right), allow users to set their **University** and **Program** (dropdowns fetching from DB) and update their `username`.

### 2. Module: Admin Dashboard (`/admin`)
**Access Control:**
* Add a boolean column `is_admin` (default false) to the `profiles` table.
* Create middleware to protect the `/admin` route. Redirect non-admins to Home.
**UI:**
* A table view of all recent Notes.
* **Actions:**
    * **"Nuke":** Delete a note instantly.
    * **"Ban User":** A button to flag a user (add `is_banned` column to profiles).
* **Stats:** Display simple cards for "Total Users" and "Total Notes".

### 3. Module: Note Creation with Markdown Compiler
**Route:** Triggered by a "+" button on the Profile or Library tab.
**UI Layout:**
* **Split Screen Editor:** Left side = Raw Text (`textarea`), Right side = Live Markdown Preview (`react-markdown`).
* **Inputs:** `Title`, `Course` (Dropdown), `Hook Summary` (Max 140 chars).

**The "Compiler" Logic (Validation):**
* Before saving to Supabase, run a client-side validation function `validateMarkdown()`.
* **Validation Rules:**
    1.  Must not be empty.
    2.  **Syntax Check:** Attempt to parse the string using a Markdown parser (like `remark` or `marked`). If the parser throws an error, alert: "Invalid Markdown Syntax."
    3.  **Structure Enforcer:** The note *must* contain at least one Header (`#`) and one list item (`-` or `1.`) to be considered a "valid study note."
* **Failure State:** Disable the "Post Note" button and show error: "Compilation Failed: Notes must follow Markdown structure."
* **Success State:** Save to `notes` table.

**Output:**
Provide the updated **SQL schema** changes (for admin/triggers) and the **React components** for the Editor and Admin Dashboard.
